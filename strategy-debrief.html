Our 4-Phase Methodology

Systematic approach to building information-dense landing pages

1

Ontological Discovery

Map the search intent landscape

Process

  • Decompose "Formbricks" into 11 fundamental query dimensions (WHO, WHAT, WHEN, WHERE, WHY, HOW, WHICH, FOR, VS, WITH, ABOUT)
  • Extract every possible entity and attribute users might search for
  • Build exhaustive taxonomies for each dimension (510+ single entities, 5,000+ 2-way combos, 50,000+ 3-way combos)
  • Validate search intent coverage using keyword research and semantic analysis

Output

  • Interactive ontology explorer showing all 11 dimensions
  • Mapping of high-intent vs low-intent combinations
  • Prioritization framework for template creation
2

Information Gap Analysis

Identify redundancy vs novelty

Competitive Intelligence

  • Audit top 10 ranking pages for each target query
  • Extract claims, data points, entities, and sources from competitors
  • Build coverage matrix showing claim frequency across competitors
  • Calculate redundancy scores (target: <70% overlap)

Nugget Gap Methodology

  • Decompose queries into atomic information needs (TREC methodology)
  • Score each nugget by competitor coverage (0-10 scale)
  • Prioritize nuggets with 0-2 competitor coverage (maximum information gain)
  • Flag saturated nuggets to minimize or skip
3

Component Architecture

Design passage-optimized sections

GEO Signal Requirements

  • Statistics density: â‰¥2 cited data points per major section (115% visibility lift proven)
  • Citation quality: >60% primary sources (not secondary/aggregator)
  • Expert quotes: â‰¥1 direct quote per 1000 words (22-37% improvement on Perplexity)
  • Passage length: 130-200 words per extractable section

Section Blueprint

  • Each section addresses a high-gain nugget with unique angle
  • Self-contained passages that can rank independently
  • No section scores 0 on unique claims (TREC nugget methodology)
  • E-E-A-T minimum: 3/5 on all dimensions (4/5 for YMYL)
4

Dual-Channel Execution

Optimize for search + LLM citation

Traditional Search Optimization

  • Topical coverage completeness vs top 10
  • Heading structure supports passage ranking
  • E-E-A-T signals present (experience, expertise, authority, trust)
  • Internal linking to strengthen topical clusters

LLM Citation Optimization

  • Cited statistics within text (not just tables)
  • Authoritative source citations embedded in passages
  • Expert quotations with attribution
  • No keyword stuffing (actively hurts LLM visibility per GEO study)
